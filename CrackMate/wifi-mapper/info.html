Project Information

This website was created as part of a university project focused on network security. Its purpose is to scan WiFi networks in an urban environment, capture handshakes where possible, and visualize the collected data on an interactive map.

WiFi networks are displayed using color codes:

  • Green: Geolocated network (secure network)
  • Orange: Handshake Captured (handshake captured, password can be cracked)
  • Red: Cracked (handshake captured and password obtained)

Each network entry contains key information:

  • BSSID (MAC address of the access point)
  • SSID (network name)
  • Signal Strength (power)
  • Channel
  • Encryption Type (e.g., WPA2, WPA3)
  • Latitude
  • Longitude
  • Location Source (if obtained during handshake capture)

Tools Used

  • Python (subprocess): used to execute system-level scanning commands such as airodump-ng and aireplay-ng.
  • TKinter: used to realize the grapichal part of the application
  • SQLite: used to store and retrieve the data about the scanned networks
  • ADB: used to retrieve the GPS position from a connected Android device during handshake capture, providing location data when available.

How It Works

1. Scanning

The tool begins by enabling monitor mode on the network interface. It scans for surrounding WiFi networks and attempts to capture WPA handshakes. Once a handshake is captured, the relevant network information is saved along with its GPS coordinates, if available via a connected Android phone and ADB.

2. Handshake Capture with GPS

If GPS coordinates are retrieved during handshake capture, an additional field named LocationSource is added to that specific networkâ€™s data. The value of this field is set to "HandshakeCapture" to indicate that the location was determined during the capture process.

3. Cracking

A dictionary-based attack is performed against the captured handshake files. The result is a list of cracked and uncracked networks, which are then displayed on an interactive map.

Data Format

The scan results are stored in a CSV file called wifi_data.csv, which acts as the application's database.

The CSV includes the following base columns:

  • bssid
  • ssid
  • power
  • channel
  • encryption
  • lat (latitude)
  • lng (longitude)

If GPS was obtained during handshake capture, a column named LocationSource is added with value "HandshakeCapture" only for those networks. The CSV will contain this column only if at least one network includes this field.
